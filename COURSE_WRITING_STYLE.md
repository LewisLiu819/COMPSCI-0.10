# 课程内容写作风格指南

本指南旨在确保所有课程内容的语言风格保持统一，使其对目标群体（4-6年级的小学生及其家长）具有吸引力、易于理解且充满趣味性。

---

## 核心原则

### 1. 目标读者
- **主要群体:** 4-6年级 (9-12岁) 的小学生。
- **次要群体:** 辅助孩子预习的家长。
- **写作要求:** 内容必须同时对孩子有吸引力，对家长清晰明了。

### 2. 整体基调
- **热情洋溢、鼓励人心、充满趣味:** 将学习过程描绘成一场“冒险”、“任务”或“挑战”。
- **避免:** 避免使用正式、学术化或枯燥的教师教学大纲语言。

### 3. 语言风格
- **直接、对话式:** 多使用第二人称（“你”、“你们”），直接与读者对话。使用第一人称复数（“我们”）来营造共同学习、共同探险的亲切感。
- **生动活泼:** 多使用感叹号和有趣的形容词来传递兴奋感。句子力求简洁明快。

---

## 具体实施指南

### 1. 标题改造
将标准的教学大纲式标题，转化为引人入胜的、提问式的或行动号召式的标题。

**英文范例:**
- `Week Overview` → `What's in Store This Week?`
- `Learning Objectives` → `What You'll Be Able to Do!`
- `Lesson Structure` → `Our Mission for Today` or `Our Investigation Plan`
- `Assessment` → `How We'll See Our Skills Grow` or `How We'll Know You're a Super Detective`
- `Materials Needed` → `Your Detective Gear` or `Your Programmer's Toolkit`

**中文范例:**
- `本周概述` → `本周有什么好玩的？`
- `学习目标` → `这周你将学会什么？`
- `课程结构` → `我们今天的任务` 或 `我们的调查计划`
- `评估` → `我们如何看到自己的进步？` 或 `如何成为超级小侦探？`
- `所需材料` → `你的侦探装备` 或 `你的程序员工具箱`

### 2. 概念简化
将复杂的专业术语，用简单、易于理解的比喻来解释。

- **算法 (Algorithm):** 解释为“一个花哨的词，意思就是步骤清单”、“就像一份食谱！”
- **调试 (Debugging):** 解释为“寻找并修复指令中的错误或问题的过程”，并将其包装成“侦探”或“修理工”的角色。
- **技巧:** 将专业术语描绘成“程序员的秘密口令”或“行话”，让学习它们变得充满仪式感和趣味性。

### 3. 活动描述
用叙事的方式生动地描述课堂活动，聚焦于**学生将要做什么和体验什么**。

- **原始:** "学生通过‘人类机器人’活动学习算法..."
- **修改后:** "变身程序员，为你专属的‘人类机器人’编写程序！你将向你的朋友下达指令，并亲眼见证为什么每一个步骤都至关重要。"

### 4. 词汇表
用儿童友好的语言来定义新词汇，而不是照搬技术定义。

- **例 (英文):** `Computer: A machine that follows instructions to help people solve problems or do tasks.`
- **例 (中文):** `算法 (Algorithm): 为完成一项任务而编写的一份分步说明清单。`

### 5. 双语一致性
- **风格统一:** 确保中英文版本在基调、风格和趣味性上保持高度一致。
- **自然本地化:** 中文版应是自然的、符合中文语境的表达，而非生硬的直译。使用能引起中国孩子和家长共鸣的词汇和说法（例如，用“寻宝游戏”来增加趣味性）。
